
# Proxies #

Sometime during your career as an /i/nsurgent, you're going to not only avoid the banhammer, but avoid getting v& as well. Proxies are servers that essentially act as buffers in between you and any sort of resource that exists on the net. Now, why is this helpful to us? Because it can help mask your identity by using the server as a middleman between you and a venue in which you wish to keep anonymous. For example, if you're going to target a forum, and would prefer to come back to troll them later (who wouldn't amirite?), you will need to use a proxy to avoid an IP ban.
In a basic HTTP format, proxies basically work by 

* you ---HTTPGET--> proxy ----HTTPGET---> target server
* target server ---webpage----> proxy ---webpage----> you
Somewhat of a stripped down illustration, but that is the basic concept of using a proxy in an http setting. Obviously, the more proxies you chain together, the more complicated (and slower) the process gets, but nevertheless the entire concept of a proxy can be demonstrated in the above example.

# Some proxies are moar equal than others #

Keep in mind that not some proxies are not as safe as others. There are certain ways using a proxy can fuck you over if you're not a little cautious. For instance, you could have your anonymity compromised if one or more of the following occur:

* Proxy is in a nation where it is more susceptible to seizure/subpoenas 
* Proxy is on a computer that is compromised
* Proxy is run by somebody who won't think twice about snooping through your traffic
Number 1 can be rectified by chaining proxies together from multiple countries, preferably ones that are hostile to each other in some form.
The threats that problems 2 and 3 bring could be mitigated by chaining proxies together and encrypting all traffic when possible.
